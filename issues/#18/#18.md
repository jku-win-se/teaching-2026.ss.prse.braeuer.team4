PMD ruleset change documentation:

- The originally provided PMD ruleset is preserved in ruleset.xml as the source specification.
- That provided file mixes Java rules with rules for other PMD languages and technologies such as Apex, ECMAScript, JSP, PLSQL, VF, VM, and XSL.
- This repository is a Java-only Maven project, so executing PMD directly against the provided ruleset caused ruleset-loading failures, for example unresolved Apex rule references.
- Maven PMD is therefore configured in pom.xml to use ruleset-effective-java.xml instead of ruleset.xml.
- ruleset-effective-java.xml is a derived Java-only subset of the provided ruleset.
- The effective file also excludes a small set of Java rule IDs that are no longer present in PMD 7, because those obsolete IDs would otherwise make the build fail before analysis can run.
- This approach preserves the provided ruleset for traceability while still enforcing the applicable Java rules in the actual build.
